**"REST API란?"**

ㅡ HTTP 통신에서 어떤 자원에 대한 CRUD 요청을 Resource와 Method로 표현하여 특정한 형태로 전달하는 방식



**Rest API의 구성 요소**

- **Resource**
  - 서버는 Unique한 ID를 가지는 Resource를 가지고 있으며, 클라이언트는 이러한 Resource에 요청을 보냅니다. 이러한 Resource는 URI에 해당합니다.
- **Method**
  - 서버에 요청을 보내기 위한 방식으로 GET, POST, PUT, DELETE가 있습니다. CRUD 연산 중에서 처리를 위한 연산에 맞는 Method를 사용하여 서버에 요청을 보내야 합니다.
- **Representation of Resource**
  - 클라이언트와 서버가 데이터를 주고받는 형태로 json, xml, text, rss 등이 있습니다. 최근에는 Key, Value를 활용하는 json을 주로 사용합니다.

##### "객체, 오브젝트, 클래스의 차이?"

- OOP에서 **오브젝트와 인스턴스**는 **실제로 존재하는 사물**을 의미
- **클래스**는 오브젝트와 인스턴스를 만들어내기 위한 **설계도**
- **객체**는 클래스로 구현한 **실체된 대상**



##### "MVC 모델이란?"

- Model, View, Controller로 나뉘어진 모델



- 1. 웹 브라우저 사용자의 요청을 Servlet이 받는다. 
  2.  Servlet은 요청을 View로 보여줄 것인지, Model로 보내줄 것인지 정하여 전송한다.  
  3.  여기서 View 페이지는 사용자에게 보여주는 역할만 담당하고 실질적인 기능의 부분은 Model에서 담당한다.

  



##### "NFC, RFID, 블루투스의 차이점 및 장단점?"

- NFC는 버스 교통카드 시스템을 예로 들 수 있으며, ㄴ

  - 가까운 거리 단말기 간 데이터를 전송하는 기술

  

- RFID는 하이패스를 예로 들 수 있으며,

  - 전파를 이용해 먼 거리에서 정보를 인식하는 기술



- 블루투스는 에어팟을 예로 들 수 있으며,
  - 초기 인증으로 안정적이다.





- **프레임워크와 라이브러리의 차이?**

  - 뼈대나 기반구조.
  - Application 개발 시 **필수적인 코드, 알고리즘 데이터베이스 연동과 같은 기능을 위해 어느정도 뼈대를 제공해주는 것**
  - **Library**는 **특정 기능에 대한 도구 or 함수를 모아둔 것**

  

- **Public, Protected, Private 에 대해 설명**

  - **캡슐화**와 **정보은닉**을 구현할 수 있다.
  - 접근 지정자.
  - public : 모든 접근 허용
  - protected : 상속 받은 클래스 또는 같은 패키지
  - private : 외부 접근 불가능

  

- **Static에 대해 설명**

  - 메모리에 한 번 할당되어 프로그램이 종료될 때 해제되는 것
  - **모든 객체가 공유하는 메모리**라는 장점.

  

- **동기화란?**

  - 여러 개의 스레드가 한 개의 자원을 사용하고자 할 때 해당 스레드만 제외하고 나머지는 접근을 못하도록 막는 것

  

- **객체지향(OOP) 개념**

  - **추상화** : 목적과 관련이 없는 부분을 제거하여 필요한 부분만을 표현하기 위한 개념

  - **캡슐화** : 외부에 노출할 필요가 없는 정보들은 은닉

  - **상속** : 코드의 중복을 없애기 위함

  - **다형성** : 형태가 같은데 다른 기능을 하는 것

    

- **블록체인이란 무엇인가?**

  - 신뢰를 만들어내는 기술

  

- **OOP 언어와 스크립트 언어 차이**

  - OOP는 객체 지향 프로그래밍
  - **스크립트언어**는 기능 중점 프로그래밍

  

- **데이터 vs 빅데이터 차이**

  - 정형, 비정형.. 데이터 마이닝

  

- **Collection이 무엇인지**

  - List, Set , Map과 같은 자주 사용하는 자료구조를 편하게 사용할 수 있도록 인터페이스로 구현해놓은 것

  

- **MVC란 무엇인가**

  - 코드 재사용성 증가를 위해 Model, View, Controller로 나눈 설계 방식

  

- **프로세스와 스레드의 차이**

  - 프로세스는 실행 가능한 명령어 집합
  - 스레드는 프로세스 내에서 실행되는 흐름

  

- **DNS에 대해**

  - 도메인에 연결된 서버 IP를 찾아주는 역할

  

- **인덱스란?**

  - **목적** : RDBMS에서 검색 속도를 높이기 위한 기술

    

- **정규화란?**

  - 이상현상을 해결하기 위해 여러 개의 릴레이션으로 분해하는 과정.



- **JVM이란?**

  - JAVA 코드를 실행하고, 어떠한 환경에서든 JAVA가 실행될 수 있도록 하는 것

  

- **추상클래스와 인터페이스의 차이**

  - 클래스 vs 명세서
  - 추상클래스는 반드시 모든 추상메소드를 구현해야 함.

  

- **Spring과 Spring boot의 차이**

  - 부트는 스프링에서 사용하는 프로젝트를 간편하게 셋업할 수 있는 서브 프로젝트. 내장 서버가 있음

  

- **IOC란?**
